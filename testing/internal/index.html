Definitions in the internal* namespaces are subject to change without notice. DO NOT USE THEM IN USER CODE!

To implement an action Foo, define: 1. a class FooAction that implements the ActionInterface interface, and 2. a factory function that creates an Action object from a const FooAction*.

The two-level delegation design follows that of Matcher, providing consistency for extension developers. It also eases ownership management as Action objects can now be copied like plain values.
