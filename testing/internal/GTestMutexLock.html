class GTestMutexLock

We cannot name this class MutexLock because the ctor declaration would conflict with a macro named MutexLock, which is defined on some platforms. That macro is used as a defensive measure to prevent against inadvertent misuses of MutexLock like "MutexLock(&mu)" rather than "MutexLock l(&mu)". Hence the typedef trick below.
