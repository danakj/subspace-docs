There is no generally useful and implementable semantics of copying a mock object, so copying a mock is usually a user error. Thus we disallow copying function mockers. If the user really wants to copy a mock object, they should implement their own copy operation, for example:

class MockFoo : public Foo { public: // Defines a copy constructor explicitly. MockFoo(const MockFoo& src) {} ... };
