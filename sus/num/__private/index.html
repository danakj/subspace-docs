one_less_than_next_power_of_two

T one_less_than_next_power_of_two(T x)

Defined at line 1026 of file intrinsics.h

Returns one less than next power of two. (For 8u8 next power of two is 8u8 and for 6u8 it is 8u8)

8u8.one_less_than_next_power_of_two() == 7 6u8.one_less_than_next_power_of_two() == 7

This method cannot overflow, as in the `next_power_of_two` overflow cases it instead ends up returning the maximum value of the type, and can return 0 for 0.

into_float_constexpr

auto into_float_constexpr( UnsafeFnMarker , T x)

Defined at line 1103 of file intrinsics.h

Prefer the non-constexpr `into_float()` to avoid problems where you get a different value in a constexpr context from a runtime context. It's safe to call this function if the argument is not a NaN. Otherwise, you must ensure the NaN is exactly in the form that would be produced in a constexpr context in order to avoid problems.

into_float

auto into_float(T x)

Defined at line 1122 of file intrinsics.h

This is NOT constexpr because it can produce different results in a constexpr context than in a runtime one. For example.

``` constexpr float x = into_float(uint32_t{0x7f800001}); const float y = into_float(uint32_t{0x7f800001}); ``` In this case `x` is `7fc00001` (the quiet bit became set), but `y` is `0x7f800001`.
