class Vec

Defined at line 52 of file vec.h

A resizeable contiguous buffer of type `T`.

Vec requires Move for its items: - They can't be references as a pointer to reference is not valid. - On realloc, items need to be moved between allocations.

reserve

public void reserve( usize additional)

Defined at line 154 of file vec.h

Reserves capacity for at least `additional` more elements to be inserted in the given Vec<T>. The collection may reserve more space to speculatively avoid frequent reallocations. After calling reserve, capacity will be greater than or equal to self.len() + additional. Does nothing if capacity is already sufficient.

The `grow_to_exact()` function is similar to std::vector::reserve(), taking a capacity instead of the number of elements to ensure space for.

# Panics Panics if the new capacity exceeds isize::MAX() bytes.

reserve_exact

public void reserve_exact( usize additional)

Defined at line 171 of file vec.h

Reserves the minimum capacity for at least `additional` more elements to be inserted in the given Vec<T>. Unlike reserve, this will not deliberately over-allocate to speculatively avoid frequent allocations. After calling reserve_exact, capacity will be greater than or equal to self.len() + additional. Does nothing if the capacity is already sufficient.

Note that the allocator may give the collection more space than it requests. Therefore, capacity can not be relied upon to be precisely minimal. Prefer reserve if future insertions are expected.

# Panics Panics if the new capacity exceeds isize::MAX bytes.

grow_to_exact

public void grow_to_exact( usize cap)

Defined at line 185 of file vec.h

Increase the capacity of the vector (the total number of elements that the vector can hold without requiring reallocation) to `cap`, if there is not already room. Does nothing if capacity is already sufficient.

This is similar to std::vector::reserve().

# Panics Panics if the new capacity exceeds isize::MAX() bytes.
